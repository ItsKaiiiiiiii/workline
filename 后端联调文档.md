# 工作流测试功能 - 后端联调文档

## 功能说明

用户配置好工作流后，点击"运行测试"，直接从头运行，每个节点自动处理输入输出，无需用户提供测试数据。

---

## 完整流程

### 1. 保存草稿

接口：`POST /api/workflow-drafts`

请求参数：
```json
{
    "draftId": null,
    "name": "测试工作流",
    "description": "",
    "definition": {
        "nodes": [...],
        "edges": [...]
    }
}
```

响应：
```json
{
    "success": true,
    "code": 200,
    "message": "Success",
    "data": {
        "draftId": "abc123def456",
        ...
    }
}
```

前端操作：保存 `data.draftId`

---

### 2. 点击"运行测试" → 直接运行！

**不需要弹出输入框！**

前端判断：
```typescript
if (!savedDraftId) {
    alert('请先保存草稿！');
    return;
}

// 直接调用测试接口
await runTest();
```

接口：`POST /api/workflow-tests/run`

请求参数：
```json
{
    "draftId": "abc123def456"
    // testInput 可选（不传也可以）
    // startNodeId 可选（不传从起点开始）
}
```

响应：
```json
{
    "success": true,
    "code": 200,
    "message": "Success",
    "data": {
        "testExecutionId": "test-exec-123",
        "status": "RUNNING",
        "nodes": [...]
    }
}
```

---

### 3. 轮询获取状态

接口：`GET /api/workflow-tests/{testExecutionId}`

轮询频率：1 秒一次

停止条件：
- `status === "COMPLETED"` ✓
- `status === "FAILED"` ✗

响应示例：
```json
{
    "success": true,
    "code": 200,
    "message": "Success",
    "data": {
        "testExecutionId": "test-exec-123",
        "status": "COMPLETED",

                "status": "COMPLETED",

            },
            {
                "nodeId": "node2",
                "nodeName": "HTTP请求",
                "nodeType": "http",
                "status": "COMPLETED",
                "inputDataSnapshot": "{\"time\":\"2024-05-02T10:00:00\"}",
```

### 4. 显示结果

- 画布上每个节点显示状态徽章
- 右侧面板显示详细信息

## 接口汇总

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/workflow-drafts` | POST | 保存草稿 |
| `/api/workflow-tests/run` | POST | 运行测试 |
| `/api/workflow-tests/{testExecutionId}` | GET | 获取执行状态 |
| `/api/workflow-tests/draft/{draftId}/latest` | GET | 获取草稿最新测试结果 |
| `/api/workflow-tests/draft/{draftId}/cleanup` | DELETE | 清理测试资源 |

## API 详细规范

### 1. 保存草稿接口

**接口地址：** `POST /api/workflow-drafts`

**请求参数：**
```typescript
interface WorkflowDraftCreateRequest {
  draftId?: string | null;
  workflowId?: string | null;
  name: string;
  description?: string;
  definition: WorkflowDefinitionData;
  savedBy?: string;
}
```

**响应格式：**
```typescript
interface WorkflowDraft {
  id: number;
  draftId: string;
  workflowId?: string | null;
  name: string;
  description?: string;
  definitionJson: string;
  createdAt: string;
  updatedAt: string;
  savedBy?: string;
}
```

---

### 2. 运行测试接口

**接口地址：** `POST /api/workflow-tests/run`

**请求参数：**
```typescript
interface RunWorkflowTestRequest {
  draftId: string;
  startNodeId?: string | null;
  testInput?: Record<string, any>;
  saveAsDefaultInput?: boolean;
}
```

**注意：**
- `testInput` 是可选的，不传也可以运行
- `startNodeId` 可选，不传从起点开始

**响应格式：**
```typescript
interface WorkflowTestExecution {
  testExecutionId: string;
  draftId: string;
  workflowId?: string | null;
  status: 'PENDING' | 'RUNNING' | 'COMPLETED' | 'FAILED';
  startedAt: string;
  endedAt?: string | null;
  durationMs?: number | null;
  testInput?: string | null;
  testOutput?: string | null;
  failedNodeId?: string | null;
  failedNodeName?: string | null;
  errorMessage?: string | null;
  errorStacktrace?: string | null;
  nodes: TestNodeExecution[];
}

interface TestNodeExecution {
  nodeId: string;
  nodeName: string;
  nodeType: string;
  nodeConfigSnapshot?: string | null;
  executionOrder: number;
  status: 'PENDING' | 'RUNNING' | 'COMPLETED' | 'FAILED';
  startedAt?: string | null;
  endedAt?: string | null;
  durationMs?: number | null;
  inputDataSnapshot?: string | null;
  outputDataSnapshot?: string | null;
  errorMessage?: string | null;
  errorStacktrace?: string | null;
}
```

---

### 3. 获取测试执行接口

**接口地址：** `GET /api/workflow-tests/{testExecutionId}`

**轮询频率：** 1 秒一次

**停止条件：** `status === 'COMPLETED'` 或 `status === 'FAILED'`

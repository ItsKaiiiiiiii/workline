# 工作流测试模块 - 联调测试指南

## 前置检查

### 1. 确认后端服务已启动

访问调试接口：
```
GET /api/workflow-tests/debug/drafts
```

如果返回 404 或连接错误，说明后端可能没有编译最新代码或服务未启动。

---

## 测试流程

### 步骤 1: 保存草稿

**操作：**
1. 在画布上添加至少一个节点
2. 填写工作流名称
3. 点击「保存草稿」按钮

**预期请求：**
```http
POST /api/workflow-drafts
Content-Type: application/json

{
  "name": "测试工作流",
  "description": "",
  "definition": {
    "nodes": [
      {
        "id": "node-xxx",
        "type": "set",
        "label": "设置变量",
        "config": {
          "result": "${body.amount * 0.9}"
        }
      }
    ],
    "edges": []
  }
}
```

**预期响应：**
```json
{
  "success": true,
  "code": 200,
  "message": "Success",
  "data": {
    "id": 1,
    "draftId": "abc123def456",
    "workflowId": null,
    "name": "测试工作流",
    "definitionJson": "{...}",
    "createdAt": "2024-05-20T10:00:00",
    "updatedAt": "2024-05-20T10:00:00",
    "savedBy": "user1"
  }
}
```

**验证要点：**
- ✅ 浏览器控制台打印「保存草稿响应」
- ✅ `data.draftId` 存在且不为 null
- ✅ 前端状态 `savedDraftId` 已更新
- ✅ `testStore.draftId` 已设置

### 步骤 4: 查看测试结果

**画布上的节点状态：**
- 🟢 绿色对勾 (COMPLETED) - 节点执行成功

**右侧测试结果面板：**
- ✅ 显示总耗时

- ✅ 显示节点列表，点击可查看详情

- ✅ 如有错误，显示错误信息和堆栈

## 常见问题排查

### 问题 1: 点击「运行测试」提示「请先保存草稿」

**排查步骤：**
1. 检查是否已点击「保存草稿」
2. 打开浏览器控制台，查看是否有「保存草稿响应」日志
3. 确认响应中 `data.draftId` 存在
4. 检查 `savedDraftId` 状态是否已更新

### 问题 2: 测试一直显示 RUNNING 状态

**排查步骤：**
1. 检查网络面板，确认轮询请求正常
2. 打开浏览器控制台，查看是否有轮询错误
3. 等待几秒，后端可能需要时间执行
4. 检查后端日志，确认测试执行是否卡住

### 问题 3: 节点执行失败

**排查步骤：**
1. 在测试结果面板点击失败的节点
2. 查看 `errorMessage` 和 `errorStacktrace`
3. 检查节点配置是否正确
4. 确认组件参数是否符合预期

### 问题 4: 保存草稿返回 "Draft not found"

**排查步骤：**
1. 检查请求中的 `draftId` 是否正确
2. 确认租户 ID（tenantId）是否匹配
3. 检查草稿是否已被删除

### 问题 5: 类型错误 - draftId 可能为 null

**说明：**
- 类型已更新为 `string | null`
- 使用可选链 `?.` 和空值合并 `??` 安全访问
- 不需要在每次使用前都检查 null

## 开发调试技巧

### 查看 Store 状态

在浏览器控制台中：
```javascript
// 查看 workflows store
window.__VUE_DEVTOOLS_GLOBAL_HOOK__.apps[0].config.globalProperties.$pinia._s.get('workflows')

// 查看 test store
window.__VUE_DEVTOOLS_GLOBAL_HOOK__.apps[0].config.globalProperties.$pinia._s.get('workflowTest')
```

### 快速测试流程

使用浏览器开发者工具的 Network 面板：
1. 保存草稿 → 复制 `draftId`
2. 运行测试 → 查看请求是否正确携带 `draftId`
3. 轮询 → 确认状态更新

### 强制清除测试状态

在浏览器控制台：
```javascript
testStore.clearTestResults()
testStore.closeTestResultPanel()
```

## 后端 API 参考

### 保存草稿
```
POST /api/workflow-drafts
```

### 运行测试
```
POST /api/workflow-tests/run
```

### 获取测试执行
```
GET /api/workflow-tests/{testExecutionId}
```

### 获取草稿的最新测试
```
GET /api/workflow-tests/draft/{draftId}/latest
```

### 清理测试资源
```
DELETE /api/workflow-tests/draft/{draftId}/cleanup
```
